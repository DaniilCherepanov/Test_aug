# Документация проекта «Читай город»

## Описание базового синтаксиса записи и форматирования

Проект использует следующие технологии и синтаксис:

1. **Python** — основной язык программирования для написания тестов.

2. **Selenium** — библиотека для автоматизации взаимодействия с веб‑браузером.

3. **Pytest** — фреймворк для написания и запуска тестов.

4. **Allure** — инструмент для генерации отчётов о выполнении тестов.

5. **Requests** — библиотека для выполнения HTTP‑запросов в API‑тестах.

6. **Python‑decouple** — библиотека для работы с конфигурационными файлами.

### Библиотеки и команды для установки:

- `pip install pytest`
- `pip install selenium`
- `pip install webdriver-manager`
- `pip install allure-pytest`
- `pip install python-decouple`
- `pip install requests`

### Форматирование кода

- Код форматируется в соответствии с PEP 8 (стиль написания кода на Python).
- Используются docstrings для документирования методов и функций.
- Все шаги теста размечаются с помощью `@allure.step` или `with allure.step` для улучшения читаемости отчетов.

## Описание тестовых сценариев:

### UI‑тесты (test_ui):
1. **test_search_books**
- Feature: Поиск книг.
- Severity: Critical.
- Цель: Проверить, что поиск книги по ключевому слову возвращает результаты.

**Шаги**:
- Открытие главной страницы.
- Выполнение поиска книги «1984».
- Проверка наличия результатов.

2. **test_add_book_to_cart**
- Feature: Добавление в корзину.
- Severity: Normal.
- Цель: Проверить возможность добавить книгу в корзину с её страницы.

**Шаги**:
- Переход на страницу книги.
- Нажатие кнопки «Купить/В корзину».

3. **test_click_cart_button**
- Feature: Работа с корзиной.
- Severity: Normal.
- Цель: Проверить переход в корзину через кнопку в верхней части сайта.

**Шаги**:
- Открытие главной страницы.
- Открытие корзины через шапку сайта.
- Ожидание загрузки корзины.

4. **test_add_book_and_go_to_cart**
- Feature: Полный сценарий покупки.
- Severity: Blocker.
- Цель: Проверить полный сценарий: добавление книги → переход в корзину → увеличение количества.

**Шаги**:
- Переход на страницу книги.
- Добавление книги в корзину.
- Переход в корзину через шапку сайта.
- Увеличение количества товара на 1.

5. **test_add_book_and_remove_from_cart**
- Feature: Управление корзиной.
- Severity: Normal.
- Цель: Проверить возможность удалить товар из корзины.

**Шаги**:
- Переход на страницу книги.
- Добавление книги в корзину.
- Переход в корзину и ожидание загрузки.
- Удаление товара из корзины.

## API‑тесты (test_api)

1. **test_search_book_by_cyrillic_name**
- Feature: API: Поиск книг.
- Severity: Critical.
- Цель: Проверить поиск книги по кириллическому названию через API.

**Шаги**:
- Отправка запроса с фразой на кириллице.
- Проверка статуса ответа 200 (OK).
- Проверка наличия поля data в ответе.

2. **test_search_book_by_latin_name**
- Feature: API: Поиск книг.
- Severity: Critical.
- Цель: Проверить поиск книги по латинскому названию через API.

**Шаги**:
- Отправка запроса с фразой на латинице.
- Проверка статуса ответа 200 (OK).
- Проверка наличия поля data в ответе.

3. **test_unauthorized_request**
- Feature: API: Поиск книг.
- Severity: Normal.
- Цель: Проверить, что публичный эндпоинт доступен без авторизации.

**Шаги**:
- Отправка запроса без токена.
- Проверка статуса 200 (OK).
- Проверка наличия данных.

4. **test_empty_search_phrase**
- Feature: API: Поиск книг.
- Severity: Normal.
- Цель: Проверить, что API корректно обрабатывает пустой поисковый запрос.

**Шаги**:
- Отправка запроса с пустой фразой.
- Проверка статуса ответа 200 (OK).
- Проверка наличия поля data в ответе.

5. **test_invalid_method**
- Feature: API: HTTP‑методы.
- Severity: Blocker.
- Цель: Проверить, что API отвергает POST‑запросы к эндпоинту поиска.

**Шаги**:
- Отправка POST‑запроса вместо GET.
- Проверка статуса ответа 405 (Method Not Allowed).
